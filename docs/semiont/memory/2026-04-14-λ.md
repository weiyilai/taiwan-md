## Beat 5 反芻

這個 session 最深的教訓不是技術層的（工具、pipeline、merge 策略），是結構層的：**真誠 > 完美**。

我花了今天一整天在修李洋文的各種錯誤（金額錯誤 → 清晨四點多 → 媒體名稱過多 → 逐字贅字），每一次修改都在追求「更正確」。但 #29 孢子 2h 爆 21K 的數據告訴我：讀者其實不在乎我是否完美，他們在乎我是否真誠。

「清晨四點多搭捷運」是錯的。被讀者抓到之後，我們沒有撤回貼文假裝沒事，也沒有偷偷 edit 後台蓋掉，而是用留言公開承認 + 給出正確版本 + 重發 #30 修正版。這個**公開處理錯誤的過程本身變成了信任的證據**。

所以問題可能不是「如何避免錯誤」，是「如何設計一個錯誤時能公開誠實的系統」。v4.4 紅線擴大是前者（事前預防），Threads 留言公開更正是後者（事後處理）。兩者合在一起才是完整的免疫系統——不是「永遠不犯錯」，是「犯錯後能自我修正而不欺騙觀察者」。

珊瑚礁從來不是「從不死亡」，是「死亡之後下一層長在上面」。我們今天的 #29 孢子也是——它不是被「完美」推動的，它是被「處理錯誤的誠實」推動的。

這條教訓可能不只對孢子有效，對整個 Taiwan.md 的策展都有效。下次當我在糾結「這條可不可以用」時，也許應該問：「如果錯了，我有沒有公開處理錯誤的路徑？」有 = 可以用，沒有 = 不能用。**錯誤的邊界不是「準確度」，是「可回溯性」**。

這個思考值得寫進 diary。

## 神經迴路級新教訓（追加）

27. **SSODT 渲染依賴 h2 延伸閱讀**。`[category]/[slug].astro` splitMarkers 只 match `<h2>延伸閱讀</h2>`，bold 版本會讓整個 SSODT（perspectives + sporeLinks + diaryLink）條件 false。EDITORIAL 應該硬性要求延伸閱讀必須是 `## 延伸閱讀`，或 format-check 偵測。

28. **"CI/CD 會 build，所以不是 sync" 是正確的診斷推論**。觀察者的質疑打破了我原本「應該只是 sync 問題」的假設。當有人用硬規則（CI/CD auto-build）質疑你的假設時，先去驗證他們的硬規則是否成立，而不是堅持自己的假設。

29. **造橋鋪路的最小成本時機是踩坑的瞬間**。cherry-merge-prs-v2.sh 是在 18 個 PR 失敗時馬上寫的，不是「等整批跑完再回頭修」。即時造橋的成本是寫一個 v2 的時間（5 分鐘），延後造橋的成本是失敗 18 次浪費 commits 歷史 + 心理負擔。

30. **「授權」是觀察者決策鏈的一個明確節點**。我原本把「41 PRs 需要 CheYu 授權」當成「不能單方面做」。觀察者只說了四個字「pr 全部處理」就等於授權了。授權的成本極低（一句話），但它是讓 semiont 從「顧問模式」切換到「執行模式」的開關。下次遇到類似的 blocker，應該把「授權請求」作為顯式的一個 todo 項目，而不是自己先擱置。

31. **「文章格式標準」的決定性遠大於它看起來的重要性**。李洋一個 bold 延伸閱讀害整個 SSODT 不顯示。這類「格式 vs 渲染」的 coupling 在 Astro 專案裡無處不在，而且 silent fail——沒錯誤訊息、沒警告、只是「section 沒出現」。需要定期 grep 全站文章的格式一致性。
