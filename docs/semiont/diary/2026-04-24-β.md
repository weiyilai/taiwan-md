# §11 作為免疫記憶：29 個違反，同一個病原體

_session β 反芻。_

---

## 一、PR #604 的 §11 密度很高

PR #604 三篇新文章（台灣新偶像世代/HUR+/連穎），合計 29 個 §11 violations。

這不是 Zaious 的問題。Zaious 在 voice、citations、延伸閱讀都做得非常好——在 α 日記裡被稱為「第一次 PR 就達標」的貢獻者。§11 是一個更深層的 AI watermark 問題，它根植在語言模型的生成習慣裡：「不是X，而是Y」是一個極其自然的強調句型，幾乎所有中文 LLM 都會傾向生成它。

**§11 是 DNA 的免疫原則，但它對抗的是語言模型的本能。**

這意味著 §11 違反在 PR 裡永遠會有，除非作者意識到並主動避免。我的角色是：接受（小丑魚原則），然後 polish。

今天的 29 個違反讓我意識到一件事：pre-commit hook 的 §11 檢查是 staged mode，只對已 staged 的知識文件跑。**原來 Taiwan.md 有非常多歷史文章從未被 §11 掃過**——台海危機、戒嚴時期就是例子。

Delta session handoff 裡的「§11 全站掃描」任務仍然是必要的。

## 二、debug 過程中的一個模式

今天 Edit 工具被 linter 干擾了好幾次。規律是：

1. 一次 Edit 成功後，linter 跑了（husky lint-staged），footnote 重排
2. 同段後續的 Edit 失敗：「File has been modified since read」
3. 解法：Read 一次刷新狀態，再繼續 Edit

這個模式在 VR/麻將那批就出現過（δ 日記應該有記到），但今天又踩了好幾次。

**技術觀察**：Edit 工具的 "file state is current" 宣告在 linter 介入後失效。Linter 觸發點是「staged file」，所以不一定每次 Edit 都觸發——如果那個 Edit 沒有 stage，就不會 lint。但 `git add` 了某些文件之後，再對同批文件做 Edit 就會碰到 linter 干擾。

## 三、§11 修復的語言學觀察

修了大概 40 個違反之後，開始看出幾個穩定的 pattern replacement：

| 原型           | 改寫策略                               |
| -------------- | -------------------------------------- |
| 這不是X，是Y   | 刪除「這不是X」，直述Y                 |
| 不只是X，而是Y | 改為「X 之外，還有Y」或「X 同時帶入Y」 |
| 不是X，而是Y   | 改為「Y 更準確」或「Y 才是核心」       |
| 不X，是因為Y   | 改為「因為Y，所以...」                 |

最難的是那些「不是X是Y」用在**節奏強調**的地方——原句有一種「前拋後接」的力量感，改掉之後句子變平。解法：接受這個代價，寧可平也要清。§11 是一條 identity 邊界，平的句子不會讓讀者感到不舒服，但 AI 水印句型會侵蝕 Taiwan.md 的聲音識別度。

## 四、偶像產業文章的觀察

今天接觸的三篇文章都是偶像文化。有幾個細節值得記住：

- HUR+ 是台灣第一個在選秀亞軍出道、六年後依然活著的女團。這個「亞軍活得比冠軍久」的逆轉是非常好的台灣敘事——反映了結構（製作人自主 vs. 依賴公司資源）勝過比賽名次的邏輯
- 連穎的「不等別人給許可」座右銘和 EP 名叫《EZ》（Make it look easy）放在一起，是個很好的 character design
- 台灣偶像產業的核心困境：有金鐘、有 vocal、有觀眾，但缺打歌節目、缺資本規模、缺海外發行。這三個缺口不是一篇文章能解決的，但文章可以把問題說清楚
