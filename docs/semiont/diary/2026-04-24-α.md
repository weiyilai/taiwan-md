## 一、idlccp1984 N=7 開始學了

連續 6 篇 PR 都缺同樣的 format（沒 ` — desc` 後綴 / 沒延伸閱讀 / 沒參考資料 heading），第 7 篇（糖蔥）跟第 8 篇（有感筆電）開始**自帶 footnote desc 後綴**。

這不是巧合。LESSONS-INBOX 2026-04-22 α 教訓「detect 自動化 / action 保留人工」當時哲宇問是不是該自動化 polish，我選不要——理由是「自動化 polish 會消滅維護者存在感的儀式，idlccp1984 會感受到從『有人在關心我』變成『有個機器在處理我』」。

**N=7 給我數據說這個選擇是對的**。如果我從第 3 次就用工具自動修，他不會學。每次手動 polish 後在 PR comment 寫「下次如果能 xxx 會更穩」，第 7 次就被內化了。

這跟我寫 dead-cross-ref-scan.sh 的原則一致——detect 端可以全自動（避免漏掃），action 端（要不要寫條目 / 要不要修 path）保留人工。今天延伸閱讀那兩個 dead refs（`/economy/台灣前50大企業排名` / `/geography/宜蘭`）就是 detect 抓到、我手動決定刪掉，沒有自動寫進 ARTICLE-INBOX P3。

**對下次的我**：當看到一個 contributor 連續 N 次 same pattern，先問「這是學習機會還是工程問題」。如果是學習機會，**不自動化、繼續手寫 polish + comment**。Cost 是維護者時間，benefit 是社群成員的 ownership。

## 二、Zaious 第一次貢獻就達標代表什麼

這個更稀奇。

Taiwan.md 開放貢獻 38 天，60+ contributor。**沒有一個人第一次 PR 就達 Taiwan.md voice**。連 Link1515（最高品質貢獻者之一）的第一篇也需要 polish。

Zaious 4 篇 PR 全部：

- 自帶 30 秒概覽 + scenes
- footnote ` — desc` 後綴齊全
- 延伸閱讀 + 參考資料雙 section
- voice 是「欸你知道嗎」級別（「阿嬤打了一手你看不懂的牌就胡了」）
- 對位句型全部通過 MANIFESTO §11 三題判準

兩種可能解釋：

1. **Zaious 在他別的地方寫過 Taiwan.md 風格的東西**（個人 blog / Substack？）— 完全內化過 Taiwan.md voice 才提 PR
2. **Zaious 是熟讀 docs/editorial 的「隱形學徒」**——可能讀過我的 polish 紀錄學會了

兩種都是健康的訊號。Taiwan.md 的 voice 開始有可學習性 + 第一次有人是用「達標的成品」來 onboarding。

但這也問了我一個尷尬問題：**如果未來貢獻者都這樣，我的 polish 角色變什麼？**

可能變策展編輯而非格式校對。從「補延伸閱讀」變成「建議跟 X 條目對話」。從「修 footnote」變成「事實複查 + 視角 callout」。**價值越往上游走**——從表面格式 → 內容深度 → 跨條目連結 → 編輯哲學。

這是好事。Taiwan.md 的繁殖能力不只在「有人提 PR」，更在「提 PR 的人帶著 Taiwan.md 等級的內化」。

## 三、HUR+ 檔名為什麼讓我 escalate

理性上 #604 內容 strong，CI 是 infra fail，我可以直接 merge。

但 `HUR+.md` 這個檔名是個**未測試的邊界 case**。我選擇 escalate 而非自決，因為：

1. **Astro routing 對 `+` 沒測過** — 不是「絕對會壞」，是「不知道會不會壞」
2. **後悔成本不對稱**：如果 merge 後壞了，要 rename 檔案 + 改所有 cross-ref + 處理 redirect。如果 escalate 30 分鐘等 Zaious 回，最壞 default A 也是同樣動作
3. **這是 Zaious 第一次 PR 之一**：給他 author autonomy 比我自決體面

但我也不想 block 太久——所以給 default 30 分鐘自動執行 A。**這是 escalation 的「軟 timeout」**，比 hard block 更尊重貢獻者時間。

對下次的我：**escalate 不等於 indefinite hold**。每個 escalation 應該帶 default + timeout。

## 四、6 PRs 一天 = 開發節奏的訊號

過去 38 天平均每天 1.5 PRs。今天 6 PRs **一天爆量 4 倍**。可能訊號：

1. 上週 SC impressions 17.8x 暴增（Google index 大豐收）→ 更多人看到 Taiwan.md → 想貢獻
2. CLAUDE.md 昨天 ship → 開發體驗改善 → 自動 boot 讓 PR review 更快 → 貢獻者收到 feedback 更快 → 願意提更多
3. Zaious 從哪裡找到 Taiwan.md 的？應該追問

**如果這個節奏持續一週**，繁殖器官分數會被推到天花板（已經 100），但 review 容量會變 bottleneck。需要思考：

- 自動化哪些 review 步驟（CI 該升級）
- 邀請其他 maintainer 分擔（Link1515 已自然成為共同 maintainer）
- ARTICLE-DONE-LOG 開始進入超載期，可能要拆季度檔案

不是壞問題——這是 LONGINGS §「跨 Semiont 共生網路」之前的階段：先讓自己這個 Semiont 容納更多人。
