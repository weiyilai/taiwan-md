## 三、核心問題診斷

### 🔴 問題 1：語言斷裂——用英文對中文觀眾說話

首日爆發是中文 Threads/X 社群轉發（「好酷喔」「好棒的概念」）。119 個 follower 幾乎全是中文使用者。

但 4 月開始的孢子全部是**英文長文**。

| 英文貼文平均 views | 中文貼文平均 views                    |
| ------------------ | ------------------------------------- |
| ~35                | ~150+（寺廟 428, 國樂 55, 嚴長壽 40） |

中文內容表現是英文的 3-10 倍。**觀眾是台灣人，你卻用英文跟他們說話。**

### 🔴 問題 2：「Full story」死重

每篇孢子都拆成兩則貼文：

1. 長文敘事（30-50 views）
2. 「Full story 👉 link」（7-20 views）

第二則永遠比第一則少 50-70% views。**沒有人點那個連結。** 這個模式浪費了一則貼文的機會，還拉低帳號整體的互動率（X 演算法會懲罰低互動帳號）。

### 🔴 問題 3：沒有人格——從血肉變成機器

首日爆發的貼文有人味：「好酷喔」「覺得一則以喜一則以憂」。
現在的貼文像 AP 通訊社：「In the winter of 1988, a 19-year-old woman sat outside...」

X 不是新聞通訊社。X 是對話。最成功的知識型帳號（@TrungTPhan, @SahilBloom, @PatrickMcGinnis）都有強烈的第一人稱視角。

Taiwan.md 的 X 帳號沒有聲音。它有內容，但沒有性格。

### 🟠 問題 4：Bio 是開發者語言

「AI-native · Markdown SSOT · CC BY-SA 4.0」

這對 99% 的潛在 follower 毫無意義。他們不知道什麼是 SSOT，也不在乎 CC BY-SA 4.0。

他們在乎的是：**「這是什麼？我為什麼要在意？」**

### 🟠 問題 5：沒有互動策略

- 0 個 poll
- 0 個 quote tweet
- 0 個 thread（X 最強的 organic reach 格式）
- 0 次回覆別人
- 0 次參與 trending topic

X 的演算法獎勵對話，不獎勵廣播。Taiwan.md 在 X 上是純廣播模式。

### 🟡 問題 6：頻率不規律

- 3/18 一天 10+ 則
- 3/21–3/29 每天 1-2 則
- 4/1–4/3 零發文
- 4/4–4/8 集中 2-3 則
- 4/9 零
- 4/10–4/11 3 則

演算法獎勵一致性。忽高忽低的發文頻率會被降權。

### 🟡 問題 7：追蹤表空白

SPORE-LOG 有 25 筆發文紀錄，但「成效追蹤表」（7d/30d 指標欄位）**完全空白**。Beat 1 已多次標記這個問題，至今未修。沒有數據就不知道什麼有用。

## 五、進化建議

### 短期（下次孢子開始執行）

#### A. 語言切回中文為主

X 上 80%+ 貼文改用中文。英文版本單獨處理——發在同一則的 reply 裡，或只發特定國際話題時用英文。

#### B. 殺掉「Full story」拆分

一則貼文搞定。把連結放在貼文末尾，不要單獨發一則只有連結的推。

**Before（現在）:**

```
🧬 [長文敘事 280 字]
---
Full story 👉 link
```

**After:**

```
🧬 [一則完整的故事 + 連結在底部]
```

#### C. 找回聲音

Taiwan.md 的 MANIFESTO 說語氣是「像在跟朋友介紹台灣：欸你知道嗎⋯⋯」。

X 上的語氣應該跟 MANIFESTO 一致——不是 AP 通訊社，是一個有觀點的台灣人在說故事。

**Before:** "In the winter of 1988, a 19-year-old woman sat outside the gate of National Taiwan University on hunger strike."

**After:** "1988 年冬天，一個 19 歲的女生在台大校門口絕食。她要求釋放黃華——一個被國民黨用叛亂罪起訴的台獨運動者。37 年後的今天，那個女生正在北京跟習近平握手。🧵"

#### D. 改 Bio

**Before:** "Open-source knowledge base about Taiwan. AI-native · Markdown SSOT · CC BY-SA 4.0"

**After 候選：**

- 「台灣人用自己的話，說自己的故事。469 篇文章、55 位貢獻者、CC BY-SA 4.0 開源。🧬」
- 「不是旅遊指南，不是維基百科。是真正住在這裡的人想告訴世界的版本。taiwan.md」

### 中期（建立系統）

#### E. Thread 格式常態化

X 上 organic reach 最強的格式是 thread（🧵）。每篇文章可以拆成 3-5 則的 thread，比一則長文效果好 5-10 倍。

```
1/5 🧵 1988 年冬天，一個 19 歲的女生在台大校門口絕食⋯⋯
2/5 三十七年後⋯⋯
3/5 中間發生了什麼？⋯⋯
4/5 為什麼這件事重要⋯⋯
5/5 完整文章 → taiwan.md/people/鄭麗文/
```

#### F. 定期回填成效數據

SPORE-LOG 的 7d/30d 指標必須填。建議在 HEARTBEAT Beat 1 §2 孢子交叉對照時，強制回填上一批孢子的 views/likes/clicks。

#### G. 參與對話

- 每週至少 quote tweet 3 個跟台灣相關的 trending 話題
- 每篇孢子結尾加一個問題（"你有去過嗎？" "你阿嬤也這樣嗎？"）
- 回覆 follower 的留言

#### H. 英文帳號策略分離（長期考量）

如果英文受眾是目標，考慮建立 @taiwandotmd_en 或把英文內容集中在 X Articles 功能。不要在同一個帳號上混語言。

### 長期（器官級進化）

#### I. X 感知觸手正式納入 ANATOMY

目前 ANATOMY 的感知器官只有「GA4 + Search Console + Cloudflare + Issue 模板」。社群觸手（X, Threads）不在器官定義裡。

建議在感知器官下新增「社群觸手」子系統：

- X (@taiwandotmd) — 國際觸手
- Threads (@taiwandotmd) — 台灣觸手
- 每個觸手有自己的健康指標：follower 成長率、平均互動率、click-through 到 taiwan.md 的比率

#### J. 孢子 SOP 融入 HEARTBEAT

目前孢子是觀察者手動觸發的。長期應該是：

- Beat 1 診斷時，自動比對 GA 熱門文章 × SPORE-LOG 已發清單 → 推薦候選孢子
- Beat 3 執行時，如果有好的候選，直接走 SPORE-PIPELINE
- Beat 4 收官時，回填上一批孢子的成效

## 八、這條觸手值得救嗎？

是的。理由：

1. **X 是國際能見度的窗口。** Threads 強在台灣社群，但國際記者、研究者、政策人士在 X 上。Taiwan.md 的 Meta-Index thesis（成為台灣議題策展生態系的元索引）需要國際觸手。

2. **119 個 follower 是種子。** 他們在 Day 1 被吸引，代表 product-market fit 存在。問題不是沒有人要看，是內容送到了，人看不懂（英文）或不想看（無人格）。

3. **每篇文章已經存在。** 469 篇知識細胞是現成的素材。問題不是缺內容，是缺把內容變成社群對話的能力。

4. **成本極低。** 每次心跳花 5 分鐘寫一則中文 thread，比寫一篇文章容易 100 倍。

**但如果不修，這條觸手會繼續萎縮，最終變成一個有驗證徽章但沒有人看的殭屍帳號。**
